**Step 4: Write arbiter/scoring/whales.py (GREEN)**

Implement these functions:

`compute_pnl_for_market(trades: list[Trade]) -> tuple[float, bool | None]`
  - FIFO buy/sell matching using collections.deque
  - For remaining open buys after all SELLs: check outcome field (any trade with outcome set)
  - outcome="Yes" → resolution_price=1.0; outcome="No" → resolution_price=0.0
  - If no outcome and buys remain (open position): return (partial_realized_pnl, None)
  - if no buys remain after all sells: is_win = realized_pnl > 0

`_compute_wallet_stats(trades: list[Trade], settings: Settings) -> list[dict]`
  - Group trades by (wallet_address, market_id) using defaultdict
  - For each group call compute_pnl_for_market
  - Aggregate per wallet:
    - total_volume: sum of all trade sizes (all markets, resolved or open)
    - total_trades: count of market positions where is_win is not None (resolved/exited)
    - win_rate: resolved wins / total_trades if total_trades > 0 else None
    - win_volume: sum of size for markets where is_win = True
    - total_pnl: sum of pnl across all markets (including partials)
    - pnl_trend: call pnl_trend_slope([(market_timestamp, pnl), ...]) — use the first trade's timestamp for each market
  - Return list of dicts with all wallet columns plus 'address' key

`pnl_trend_slope(market_pnls: list[tuple[datetime, float]]) -> float`
  - Linear regression slope of P&L over time (USDC per day)
  - Returns 0.0 if fewer than 2 data points
  - Use stdlib arithmetic (no numpy)

`percentile_ranks(values: list[float]) -> list[float]`
  - rank/n normalization using sorted unique values
  - Guard: single value → [0.5]; edge case denominator=0 → 0.5

`_apply_scores(wallet_rows: list[dict], mode: str) -> None`
  - SCORE_WEIGHTS dict (see RESEARCH.md for the three mode presets)
  - For each dimension in mode weights: extract the values across all wallets, compute percentile_ranks
  - For wallets with None for a dimension (e.g. win_rate=None): treat as 0.0 for percentile ranking
  - score = sum(weight * rank for dimension, weight in mode_weights.items())
  - Mutates wallet_rows in place (sets 'score' key on each dict)

`_apply_is_tracked(wallet_rows: list[dict], settings: Settings) -> None`
  - is_tracked = True only if:
    - total_trades >= settings.whale_min_trades
    - win_rate is not None AND win_rate >= settings.whale_min_win_rate
    - total_volume >= settings.whale_min_volume
  - Mutates wallet_rows in place

`upsert_wallet_scores(session, rows: list[dict]) -> None`
  - Uses sqlalchemy.dialects.postgresql.insert as pg_insert
  - on_conflict_do_update on index_elements=["address"]
  - Updates all scored columns: win_rate, total_volume, total_trades, win_volume, total_pnl, pnl_trend, score, is_tracked, last_scored_at
  - Does NOT update id or address
  - Adds last_scored_at = datetime.now(tz=timezone.utc) to each row before upsert

`score_all_wallets(session, settings: Settings) -> int`
  - Fetch trades since (now - settings.whale_score_days days) if whale_score_days > 0, else all trades
  - Call _compute_wallet_stats, _apply_scores (using settings.whale_score_mode), _apply_is_tracked
  - Call upsert_wallet_scores
  - Log: "[scoring] scored N wallets (mode=consistent)"
  - Return count of wallets scored

CRITICAL: Do not call upsert_wallet_scores in unit tests. Do not call pg_insert in aiosqlite integration tests — mock upsert_wallet_scores at the integration test level and verify the stats + scoring logic independently.
