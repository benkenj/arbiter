Implement the PostgreSQL database layer: SQLAlchemy 2.0 ORM models for all three tables, an async session factory, and Alembic migrations with the initial schema including the partial unique index for signal deduplication.

from sqlalchemy.orm import DeclarativeBase
class Base(DeclarativeBase):

```

- description: Mapped[Optional[str]] = mapped_column(Text, nullable=True)
- end_date: Mapped[Optional[datetime]] = mapped_column(DateTime(timezone=True), nullable=True)
- resolved: Mapped[bool] = mapped_column(Boolean, nullable=False, server_default="false")
- closed: Mapped[bool] = mapped_column(Boolean, nullable=False, server_default="false")
- yes_price: Mapped[Optional[float]] = mapped_column(Float, nullable=True)

- __table_args__: include Index("ix_markets_active", "active")

Model: Signal
- id: Mapped[int] = mapped_column(primary_key=True)
- market_id: Mapped[int] = mapped_column(Integer, ForeignKey("markets.id"), nullable=False)
- market_question: Mapped[str] = mapped_column(Text, nullable=False)  # denormalized cache
- strategy: Mapped[str] = mapped_column(String(50), nullable=False)  # 'longshot_bias' | 'time_decay'
- signal_direction: Mapped[str] = mapped_column(String(10), nullable=False)  # 'yes' | 'no'
- signal_price: Mapped[float] = mapped_column(Float, nullable=False)
- hours_to_expiry: Mapped[Optional[float]] = mapped_column(Float, nullable=True)
- liquidity_at_signal: Mapped[Optional[float]] = mapped_column(Float, nullable=True)
- status: Mapped[str] = mapped_column(
    Enum("active", "resolved_correct", "resolved_incorrect", "expired", "void", name="signal_status"),
    nullable=False,
    server_default="active",
  )
- resolution_outcome: Mapped[Optional[str]] = mapped_column(String(20), nullable=True)  # 'YES'|'NO'|'NA'|'DISPUTED'
- fired_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), nullable=False)
- resolved_at: Mapped[Optional[datetime]] = mapped_column(DateTime(timezone=True), nullable=True)
- __tablename__ = "signals"
- __table_args__: include the partial unique index:

      "market_id",
      "strategy",
      unique=True,

Create arbiter/db/session.py with:
- make_engine(database_url: str) -> AsyncEngine: creates engine with pool_size=5, max_overflow=10, pool_pre_ping=True, echo=False

Do NOT create a module-level engine or session. Both functions take explicit parameters so callers (main.py) control the lifecycle. This avoids import-time config dependency.

```

Create the initial migration MANUALLY (do NOT use --autogenerate for this migration — the partial index is not reliably generated):
```bash
alembic revision -m "initial_schema"
```

Then edit the generated file at alembic/versions/xxxx_initial_schema.py (rename conceptually to 001). Fill in upgrade() and downgrade() with the exact SQL operations:

upgrade():
1. Create enum type: op.execute("CREATE TYPE signal_status AS ENUM ('active', 'resolved_correct', 'resolved_incorrect', 'expired', 'void')")
2. Create markets table with all columns from the ORM model (use op.create_table)
3. Create ix_markets_active index: op.create_index("ix_markets_active", "markets", ["active"])
4. Create signals table with all columns (status column uses server_default='active')
5. Create the partial unique index MANUALLY:
   op.create_index(
       "ix_signals_market_strategy_active",
       "signals",
       ["market_id", "strategy"],
       unique=True,
       postgresql_where="status = 'active'",
   )
6. Create price_snapshots table
7. Create ix_price_snapshots_market_fetched: op.create_index("ix_price_snapshots_market_fetched", "price_snapshots", ["market_id", "fetched_at"])

downgrade():
Drop in reverse: drop price_snapshots, drop index on signals, drop signals, drop markets, drop enum type.

Do NOT use --autogenerate for this file. Write all op.create_table calls explicitly using sa.Column() syntax (not Mapped — migrations use direct Column declarations). Match column names and types exactly to what the ORM models declare.

After writing the file, verify with:
```bash

```
