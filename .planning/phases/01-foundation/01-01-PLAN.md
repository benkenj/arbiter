Add a Pydantic field_validator on database_url that checks the string starts with "postgresql+asyncpg://" and raises ValueError with message: "Must use asyncpg dialect: postgresql+asyncpg://user:pass@host/dbname"

implement load_settings() -> Settings:
  try:
    return Settings()
  except ValidationError as exc:
    print("Configuration errors — fix all before starting:\n", file=sys.stderr)
    for err in exc.errors():
      field = str(err["loc"][0])
      env_var = field.upper()  # pydantic-settings maps field name to env var by uppercasing
      msg = err["msg"]
      field_info = Settings.model_fields.get(field)
      hint = field_info.description if field_info else ""
      print(f"  {env_var}: {msg}", file=sys.stderr)
      if hint:
        print(f"    Hint: {hint}", file=sys.stderr)
    sys.exit(1)

implement print_config_summary(settings: Settings) -> None using logging.info:
  Group output in three sections:
  - "=== Database ===" with DATABASE_URL masked: show first 30 chars + "..."
  - "=== Detection Thresholds ===" with all detection fields
  - "=== Logging ===" with LOG_LEVEL

Use model_config = SettingsConfigDict(env_file=".env", env_file_encoding="utf-8", extra="ignore")

Do NOT use python-dotenv directly — pydantic-settings handles .env loading internally.
