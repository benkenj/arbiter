  <name>Task 2: Implement discovery loop with filter, upsert, and heartbeat</name>
  <files>arbiter/discovery/__init__.py, arbiter/discovery/loop.py</files>
  <action>
Create `arbiter/discovery/__init__.py` as an empty file.

Create `arbiter/discovery/loop.py` with the following components:

**`_is_binary(market) -> bool`:**
Check that `[o.lower() for o in (market.outcomes or [])] == ["yes", "no"]`. This rejects multi-team 2-option markets (e.g. `["Team A", "Team B"]`) and non-standard outcome labels. Return False if outcomes is empty.

**`_apply_filters(markets, settings) -> tuple[list[Market], int]`:**
Iterate over markets, applying three filters in order:
1. If `settings.market_binary_only` and not `_is_binary(m)` → skip (filtered_out++)
2. If `(m.volume or 0) < settings.market_min_volume` → skip (filtered_out++). Use `(m.volume or 0)` to handle None as 0.
3. If `(m.liquidity or 0) < settings.market_min_liquidity` → skip (filtered_out++). Use `(m.liquidity or 0)` for None.
Return `(passing_markets, filtered_out_count)`.

**`_to_db_row(market) -> dict`:**
Convert a `clients.polymarket.Market` pydantic model to a dict matching the `db.models.Market` columns:
```python
{
    "external_id": market.id,
    "question": market.question,
    "description": market.description,
    "end_date": _parse_end_date(market.end_date),  # see below
    "resolved": market.resolved,
    "closed": market.closed,
    "yes_price": market.yes_price,
    "liquidity": market.liquidity,
    "volume": market.volume,
    "active": True,
    "condition_id": market.condition_id,
    "fetched_at": datetime.now(timezone.utc),
    # created_at is intentionally NOT set here — only set on insert, not on update
}
```

`_parse_end_date(end_date_str)`: Convert ISO string to timezone-aware datetime, or return None. Use `datetime.fromisoformat(end_date_str).replace(tzinfo=timezone.utc)` if string lacks tzinfo; catch ValueError and return None.

**`upsert_markets(session, market_rows: list[dict]) -> int`:**
Use `sqlalchemy.dialects.postgresql.insert(Market)`. Conflict target: `index_elements=["external_id"]`.

On conflict, update all mutable fields (question, description, end_date, resolved, closed, yes_price, liquidity, volume, active, condition_id, fetched_at). Do NOT update `created_at` on conflict — it should be set only on the initial insert.

To set `created_at` on first insert only: add `"created_at": datetime.now(timezone.utc)` to the values dict in `_to_db_row()`, but use `postgresql.insert` with `set_={}` that explicitly excludes `created_at` from the UPDATE list (by not including it in the SET clause). The INSERT sets it; the UPDATE preserves the existing value.

Wait — the values dict already has `created_at` in the INSERT portion. The conflict SET clause simply omits `created_at`, so existing rows keep their original `created_at`. New rows get the value from the INSERT.

Update `_to_db_row()` to include `"created_at": datetime.now(timezone.utc)` in the returned dict.

Await `session.execute(stmt)` and `await session.commit()`. Return `len(market_rows)` as upserted count.

**`run_discovery_cycle(settings, session_factory, client) -> tuple[int, int, int]`:**
1. `all_markets = await client.fetch_all_active_markets()` (tenacity retry is already on `_fetch_page` inside)
2. `passing, filtered_out = _apply_filters(all_markets, settings)`
3. Convert to rows: `market_rows = [_to_db_row(m) for m in passing]`
4. Open session: `async with session_factory() as session: upserted = await upsert_markets(session, market_rows)`
5. Count new markets: query `SELECT COUNT(*) FROM markets WHERE created_at > cycle_start_time` using a separate session. Pass `cycle_start` (measured before step 1) into this function.
   - Actually: simplify by querying `SELECT COUNT(*) FROM markets WHERE created_at >= :t` where `t` is a datetime slightly before the cycle. Pass `cycle_start_dt = datetime.now(timezone.utc)` as parameter into `run_discovery_cycle`.
   - Revise signature: `run_discovery_cycle(settings, session_factory, client, cycle_start_dt: datetime)`
6. Return `(upserted, new_count, filtered_out)`

**`discovery_loop(settings, session_factory, client) -> None`:**
```python
async def discovery_loop(settings, session_factory, client) -> None:
    while True:
        t0 = time.monotonic()
        cycle_start_dt = datetime.now(timezone.utc)
        try:
            upserted, new_count, filtered = await run_discovery_cycle(
                settings, session_factory, client, cycle_start_dt
            )
            elapsed = time.monotonic() - t0
            logger.info(
                "[discovery] cycle complete in %.1fs — %d upserted, %d new, %d filtered out",
                elapsed, upserted, new_count, filtered,
            )
        except Exception as exc:
            elapsed = time.monotonic() - t0
            logger.error("[discovery] cycle failed after %.1fs: %s", elapsed, exc)
        await asyncio.sleep(settings.discovery_interval_seconds)
```

Key notes:
- Do NOT catch `BaseException` — only `except Exception` so that `asyncio.CancelledError` (a `BaseException`) propagates on SIGTERM for clean shutdown.
- Sleep goes AFTER the cycle (not before) — first cycle runs immediately on entry per user decision.
- One session per cycle, not per market row — upsert all rows in one statement.
- The `fetched_at` datetime must use `datetime.now(timezone.utc)` — never `datetime.utcnow()` (deprecated in Python 3.12).
